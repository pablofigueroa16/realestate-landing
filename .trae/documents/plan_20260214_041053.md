I will fix the "window is not defined" error in `src/components/UnitTypesSection.tsx`.

1.  **Issue**: The code `transform: translateX(-${currentIndex * (100 / itemsPerView + (32 / window.innerWidth * 100))}%)` accesses `window.innerWidth` during rendering, which fails on the server side (SSR) where `window` is undefined.

2.  **Solution**: Replace the JavaScript-based gap calculation with a pure CSS `calc()` expression.
    *   The formula `100% / itemsPerView` gives the width of one item (including its share of the gap if using flex basis logic, or excluding if using margins).
    *   The simplified logic I derived is: `calc(-1 * currentIndex * (100% + 32px) / itemsPerView)`.
    *   This works because `100%` inside `translateX` refers to the container width. `32px` is the gap (`gap-8`).
    *   So, shifting by `(100% + 32px) / itemsPerView` moves exactly one "slide width + one gap width" to the left.

3.  **Implementation**:
    *   Modify the `style` prop of the carousel track div.
    *   Remove `window.innerWidth` usage entirely.
    *   Use `style={{ transform: \`translateX(calc(-${currentIndex} * (100% + 32px) / ${itemsPerView}))\` }}`.

This ensures the component renders correctly on both server and client without hydration errors or runtime crashes.
